Sockets Overview

A construct that allows processes to communicate (typically different systems across a network). A socket in Unix based systems is basically a file descriptor, that is, an integer assigned with some open file on a system. Typically, this is to provide some level of I/O. You can use send and recv to handle interactions with sockets, but because they are the same as files, you can also use read() and write().

Typical Types of Sockets

Stream & Datagram

Protocol differences:

  • UDP (User Datagram Protocol): A connectionless protocol, you don't first establish an e2e connection, you just send data to an IP/Socket and hope that they get to the right place.

  • TCP (Transmission Control Protocol): Connection oriented. First the endpoints establish a connection, and then data is transferred back and forth.
